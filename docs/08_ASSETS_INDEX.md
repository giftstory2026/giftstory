# Gift Story — Assets Index

## 1. Purpose of This File

This file defines how Gift Story visual assets should be organized, named, understood, reviewed, and used.

Its purpose is to prevent confusion between:

* Digital assets.
* Print-ready assets.
* Preview images.
* Source files.
* Packaging files.
* Product photos.
* Mockups.
* Archived files.

Gift Story assets should always be easy to locate, easy to preview, and easy to use without guessing.

---

## 2. Core Asset Rule

Every important asset should include three parts:

1. **Source File**
   Editable or production file, such as AI, SVG, PDF, PSD, or Canva export.

2. **Preview File**
   A PNG or JPG image that clearly shows the asset visually.

3. **Usage Note**
   A short explanation of where the asset should be used and where it should not be used.

This is especially important for AI/vector files because they may not always be visually readable without design software.

## 4. File Naming Rules

All asset filenames should be in English.

General naming format:

```text id="4i3r5m"
GS_AssetType_Description_Use_Version.ext
```

Examples:

```text id="meqjsw"
GS_Logo_Primary_Gold_Print_v1.ai
GS_Logo_Primary_Digital_Preview_v1.png
GS_Castle_Interior_Gold_Print_v1.ai
GS_Pattern_Stars_Moon_Tissue_v1.svg
GS_Sticker_Round_Castle_5cm_v1.pdf
GS_Card_Gift_Front_Back_v1.ai
GS_Box_Inner_40x30x10_Mockup_v1.png
GS_Bag_Outer_45x33x15_Mockup_v1.png
```

Avoid:

```text id="7nt4or"
final logo new latest 2.ai
gift story copy.pdf
new design.png
last version final final.ai
image123.png
```

---

## 5. Asset Status Labels

Each asset should have a clear status.

Recommended status labels:

| Status         | Meaning                                |
| -------------- | -------------------------------------- |
| `Approved`     | Ready to use                           |
| `Print Ready`  | Ready for production / supplier        |
| `Digital Only` | For website/social use only            |
| `Preview Only` | Visual reference, not production-ready |
| `Needs Export` | Source exists but needs proper export  |
| `Needs Review` | Requires approval before use           |
| `Archived`     | Old reference only                     |

No asset should be used for print unless marked as `Print Ready`.

## 6. Digital vs Print Assets

Gift Story uses two asset types:

### Digital Assets

Used for:

* Official website.
* Social media.
* Instagram.
* Reels.
* Digital mockups.
* Email visuals.
* Customer previews.
* Brand presentations.

Digital assets may include:

* Soft pink heart detail.
* Light background gradients.
* Soft shadows.
* Digital texture.
* Decorative effects if subtle.

---

### Print Assets

Used for:

* Outer gift bag.
* Inner box.
* Sticker.
* Gift card.
* Tissue paper.
* Packaging print.
* Supplier production.

Print assets must follow:

* One-color muted / satin gold printing.
* Vector or high-resolution production format.
* No gradients.
* No shadows.
* No low-resolution raster elements.
* No unnecessary colors.
* No very thin lines.
* No hidden elements.
* Clear margins and safe areas where needed.

## 7. Logo Assets

Folder:

```text id="7y34xh"
assets/logo/
```

The logo is the primary brand identifier.

It may be used on:

* Official website.
* Outer gift bag.
* Inner box exterior.
* Social media.
* Email signature.
* Product photos.
* Packaging mockups.
* Sales platform.

Recommended files:

```text id="eo8p5u"
assets/logo/
  GS_Logo_Primary_Digital.svg
  GS_Logo_Primary_Digital.png
  GS_Logo_Primary_Digital_Preview.png
  GS_Logo_Primary_Gold_Print.ai
  GS_Logo_Primary_Gold_Print.pdf
  GS_Logo_Primary_Gold_Print_Preview.png
```

Usage rules:

* Use digital version for website and social.
* Use gold print version for packaging.
* Pink heart may appear in digital assets.
* Printed logo should use unified muted / satin gold.
* Do not stretch, compress, or rotate the logo.
* Do not place logo on crowded backgrounds.
* Do not use low-resolution logo files for print.

Usage rules:

* Sticker should not be crowded.
* Important elements must stay within safe area.
* Print file should include bleed if required.

Main packaging dimensions:

```text id="flbyrj"

```

Recommended files:

```text id="knagkt"
assets/packaging/
  GS_Box_Inner_40x30x10_Exterior.ai

  GS_Packaging_Full_Presentation_Mockup.png
```

Usage rules:

* Outer gift bag is part of the branded gift presentation.
* Protective nylon or shipping carton is logistics protection only.
* Shipping labels should not be placed on branded packaging.
* Packaging mockups should be visually clean and realistic.
* Do not mix protective packaging with luxury identity visuals.

---

## 18. Archive Rule

Old, unclear, duplicate, or outdated files should be moved to:

```text id="07jh8e"
archive/old-assets/
```

Archive files are not deleted, but they are not treated as approved source files.

Use archive only for:

* Historical reference.
* Recovering old ideas.
* Comparing previous versions.
* Checking original concepts.

Do not use archive files directly in the official website, production, or customer-facing material unless reviewed and re-approved.

---

## 19. Asset Review Checklist

Before approving any asset, check:

* Is the file name clear?
* Is the folder correct?
* Is there a preview image?
* Is the source file available?
* Is it digital or print?
* Is it approved?
* Is it high quality?
* Does it match the brand identity?
* Does it follow the one-color print rule if physical?
* Does it include unnecessary colors?
* Does it have enough spacing?
* Does it look premium?
* Does it create confusion?
* Is it ready for the intended use?

## 21. What ChatGPT / AI Can Understand Best

For AI-assisted work, the best format is:

1. Clear PNG preview.
2. PDF preview if there are multiple pages.
3. SVG if available.
4. Short description of the asset.
5. Correct file name.
6. Correct folder path.
7. Usage note.

AI may not reliably read or interpret AI files directly unless a preview is provided.

Therefore, every final AI/vector asset should also include:

```text id="n0wooq"
-preview.png
```

or

```text id="imuyuj"
-preview.pdf
```

This helps future AI support understand the asset quickly.

## 23. Website Asset Requirements

Website-ready assets should usually be exported as:

* SVG for logos and simple icons.
* WebP for photos.
* PNG for transparent graphics.
* JPG for compressed photography when suitable.
* MP4 / WebM for website video if needed.

Website assets should be optimized for:

* Mobile loading.
* Clean visual quality.
* Correct dimensions.
* SEO-friendly filenames.
* No unnecessary large file sizes.

---

## 24. Print Asset Requirements

Print-ready assets should usually be delivered as:

* AI.
* PDF Vector.
* EPS if needed.
* SVG when suitable.

Print files should:

* Use one print color unless approved.
* Convert fonts to outlines.
* Include correct dimensions.
* Include bleed and safe area where required.
* Avoid low-resolution images.
* Remove hidden unused layers.
* Be reviewed with supplier before production.
* Include preview PDF or PNG.

## 26. Customer-Facing Image Rules

Customer-facing product images should not show:

* Price tags.
* Discount stickers.
* Retail store names.
* Ross / outlet labels.
* MSRP.
* Dollar signs.
* Purchase information.
* Messy backgrounds.
* Unedited raw product shots.
* Poor lighting.
* Personal information.
* Supplier screenshots.

Customer-facing images should show:

* Clean product.
* Soft background.
* Clear shape.
* Brand-appropriate styling.
* Premium presentation.
* No unnecessary text.

---

## 27. Internal Reference Image Rules

Internal reference images may include:

* Purchase source.
* Cost tags.
* Product labels.
* Size tags.
* SKU.
* Retail comparison.
* Supplier screenshots.
* Condition details.

These should be kept separate from customer-facing assets.

Recommended folder:

```text id="w191wi"
assets/product-photos/[category]/[drop]/raw/
```

or inventory reference folders.

---

## 28. Assets and Inventory Connection

Every product asset should connect to inventory.

Recommended link fields in inventory:

* Product image filename.
* Product folder path.
* Client display image filename.
* Raw image filename.
* Product cost.
* Category.
* Suggested box.
* Status.
* Notes.

This prevents losing track of which product belongs to which drop or box.

---

## 29. Assets and Website Connection

Every website asset should have a defined website use.

Examples:

| Asset                  | Website Use                      |
| ---------------------- | -------------------------------- |
| Logo                   | Header, footer                   |
| Castle                 | Brand story, packaging page      |
| Pattern                | Background or divider            |
| Packaging mockup       | Packaging page                   |
| Product image          | Current story / previous stories |
| Category card image    | Coming Soon cards                |
| Special request visual | Special Request page             |

No random asset should be placed on the website without a clear role.

---

## 30. Assets and Packaging Connection

Packaging assets should map clearly to physical components.

| Asset              | Packaging Use                   |
| ------------------ | ------------------------------- |
| Logo print file    | Outer bag and box exterior      |
| Castle print file  | Inner box interior              |
| Pattern print file | Tissue paper                    |
| Sticker print file | Round closure sticker           |
| Gift card file     | Message card                    |
| Packaging mockup   | Supplier and approval reference |

Each physical packaging element should have a matching production file and preview.

## 32. Final Asset Decision

Gift Story assets must be organized as a brand system.

Every important asset should have:

* Clear English file name.
* Correct folder.
* Source file.
* Preview file.
* Digital or print status.
* Usage rule.
* Approval status.

The goal is to make Gift Story easy to design, update, print, photograph, and expand without losing consistency.

Assets are not random images.

They are the visual memory of the Gift Story world.

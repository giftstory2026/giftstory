# Gift Story — Official Website Structure

## 1. Purpose of This File

This file defines the official website structure for Gift Story.

Its purpose is to separate the brand website from the commerce platform and to make sure the website presents Gift Story as a premium gifting brand, not as a basic one-page checkout.

The official website should build trust, desire, and brand value before sending customers to the active sales platform.

---

## 2. Website Role

The official Gift Story website is the brand home.

Recommended domain:

```text
gift-story.com
```

The website should be used for:

* Brand presentation.
* Storytelling.
* Visual identity.
* Gift categories.
* Packaging experience.
* Current story teaser.
* Previous or sample stories.
* Special Gift Story Request.
* Policies.
* Contact and inquiries.
* Directing customers to the sales platform.

The website should not be used as the main operational checkout unless a full custom commerce system is built later.

---

## 3. What the Website Should Not Be

The official website should not be:

* A basic one-page landing page only.
* A crowded product catalog.
* A replacement for Salla or Zid.
* A public WhatsApp ordering page.
* A discount-style store.
* A place where all channel prices are shown.
* A platform for uncontrolled custom orders.
* A page full of unfinished promises.

The website must look calm, premium, organized, and intentional.

---

## 4. Website and Commerce Separation

Gift Story should use a two-layer structure:

### Layer 1 — Official Website

Purpose:

* Brand trust.
* Storytelling.
* Visual experience.
* Category introduction.
* Packaging explanation.
* Special request form.
* SEO.
* Customer education.

### Layer 2 — Commerce Platform

Purpose:

* Product listings.
* Prices.
* Inventory.
* Checkout.
* Payment.
* Orders.
* Invoices.
* Delivery settings.

This separation protects the premium brand feeling while keeping operations easy to manage.

## 5. Main Website CTA

The main shopping CTA should be:

**English:** Shop The Current Story
**Arabic:** تسوّقي القصة الحالية

## 17. Policies Page

Page path:

```text
/policies
```

### Purpose

Clarify customer expectations and protect the brand operationally.

### Page Should Include

* Return and exchange policy.
* Preparation time.
* Delivery and shipping.
* Special requests.
* Color and print variation note.
* Packaging protection.
* Order cancellation rules.
* Customer data and privacy note if needed.

### Key Policy Notes

#### Return and Exchange

Due to the nature of gift items, baby/kids products, and prepared packaging, returns and exchanges may not be available after the order leaves preparation, except where required or when there is a clear fulfillment error.

#### Print and Color Variation

**Arabic:**

## 21. Website Pricing Rule

The official website should avoid showing detailed active product prices.

Reason:

Different platforms may have different prices.

Examples:

* Salla / Zid price.
* HungerStation price.
* ToYou price.
* The Chefz price.
* Special request quote.

Prices should live inside each sales channel.

The official website should focus on value, story, and presentation.

---

## 22. Visual Direction

The website should visually feel:

* Off-white.
* Creamy.
* Nude.
* Muted gold.
* Soft.
* Spacious.
* Premium.
* Story-like.
* Clean.
* Warm.

Avoid:

* Dark navy as the main identity.
* Crowded product grids.
* Cheap banners.
* Heavy icons.
* Excessive animations.
* Random stock images.
* Too much text on the homepage.
* Direct discount language.

---

## 23. Image Strategy

The website should use:

* Real packaging photos when available.
* Clean mockups when real photos are not ready.
* Soft product styling.
* Flat lay images.
* Close-up packaging details.
* Gift card details.
* Tissue and sticker shots.
* Category cards.
* Previous story visuals.

Avoid:

* Raw supplier photos.
* Price tags.
* Store tags.
* Discount labels.
* Low-quality screenshots.
* Crowded backgrounds.
* Photos that show purchase source or retail stickers.

## 24. Mobile Experience

The website must work well on mobile.

Mobile priorities:

* Fast loading.
* Clear CTA.
* Easy navigation.
* Large readable text.
* Strong first impression.
* Sticky Shop The Current Story button if needed.
* Simple Special Request form.
* No heavy clutter.

Most customers will likely view the site from mobile, so mobile design should be considered first.

* Clearly explains the brand.

Gift Story’s official website should be a premium brand website, not a simple one-page sales page.

**Shop The Current Story**

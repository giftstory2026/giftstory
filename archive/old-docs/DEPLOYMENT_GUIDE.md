# Gift Story — دليل رفع الموقع على GitHub و Cloudflare Pages

## 1. الهدف

هذا الملف يوضح طريقة رفع مشروع Gift Story على GitHub ثم ربطه مع Cloudflare Pages لتشغيل الموقع على الدومين الرسمي.

---

## 2. الملفات المطلوبة

داخل المشروع يجب أن تكون الملفات الأساسية بهذا الشكل:

```text
giftstory/
  index.html
  README.md
  assets/
  docs/
```

أهم ملف للتشغيل هو:

```text
index.html
```

---

## 3. رفع الملفات على GitHub

### الخطوة 1

افتح مستودع GitHub الخاص بالمشروع:

```text
giftstory2026/giftstory
```

### الخطوة 2

ارفع محتويات مجلد المشروع، وليس ملف ZIP نفسه.

يجب أن يظهر ملف `index.html` مباشرة في الصفحة الرئيسية للمستودع، وليس داخل مجلد فرعي زائد.

الصحيح:

```text
giftstory/index.html
```

غير الصحيح:

```text
giftstory/giftstory_project/index.html
```

---

## 4. الربط مع Cloudflare Pages

### الخطوة 1

ادخل إلى Cloudflare Dashboard.

### الخطوة 2

افتح:

```text
Workers & Pages
```

ثم اختر:

```text
Create application
```

ثم:

```text
Pages
```

ثم:

```text
Connect to Git
```

### الخطوة 3

اختر مستودع GitHub:

```text
giftstory2026/giftstory
```

### الخطوة 4

إعدادات البناء:

```text
Framework preset: None
Build command: اتركه فارغًا
Build output directory: /
Root directory: /
```

إذا طلب Cloudflare مجلد النشر، اجعله الجذر `/` لأن الموقع عبارة عن ملف HTML ثابت.

---

## 5. ربط الدومين

بعد نجاح النشر، اربط الدومين:

```text
gift-story.com
```

من إعدادات Cloudflare Pages:

```text
Custom domains
```

ثم أضف:

```text
gift-story.com
www.gift-story.com
```

---

## 6. فحص الموقع بعد النشر

بعد نشر الموقع، راجع التالي:

- فتح الموقع على الجوال.
- فتح الموقع على الكمبيوتر.
- زر السلة يعمل.
- زر واتساب يفتح الرقم الصحيح.
- السعر 299 ريال.
- المقاس 40×30×10 سم.
- السياسة ظاهرة.
- لا يوجد رقم واتساب تجريبي.
- لا توجد صور أو روابط مكسورة.

---

## 7. التحديثات المستقبلية

أي تعديل لاحق على `index.html` يتم بهذه الطريقة:

1. تعديل الملف في VS Code.
2. رفع التعديل إلى GitHub.
3. Cloudflare Pages ينشر التحديث تلقائيًا.

---

## 8. ملاحظات مهمة

- لا ترفع ملفات AI الكبيرة داخل الموقع إلا إذا كانت مطلوبة للأرشفة.
- يفضل وضع صور الموقع النهائية بصيغة WebP أو PNG مضغوط.
- استخدم SVG للشعار والقلعة إذا أمكن.
- لا تستخدم ملفات PDF كشعارات داخل الموقع.
- احتفظ بملفات الطباعة داخل مجلد منفصل أو خارج نسخة الموقع إذا كانت كبيرة.

---

## 9. قائمة فحص قبل الإطلاق الرسمي

- [ ] الشعار النهائي مضاف.
- [ ] القلعة النهائية مضافة.
- [ ] صور المنتج الحقيقية مضافة.
- [ ] رسالة واتساب صحيحة.
- [ ] السياسة واضحة.
- [ ] الموقع سريع على الجوال.
- [ ] الدومين يعمل.
- [ ] رابط www يعمل.
- [ ] لا توجد بيانات تجريبية.
- [ ] تمت تجربة الطلب من الجوال.

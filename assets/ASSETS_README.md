# Gift Story — Assets README

## 1. Purpose

This folder contains the visual assets for Gift Story.

Assets must be organized clearly so they can be used for:

* Website design.
* Packaging production.
* Social media.
* Product photography.
* Mockups.
* Print files.
* AI-assisted review and future updates.

No important asset should be stored without a clear folder, file name, and preview.

---

## 2. Main Rule

Every important asset should include:

1. **Source file**
   Editable or production file such as AI, SVG, PDF, PSD, or Canva export.

2. **Preview file**
   PNG or JPG image that clearly shows the asset visually.

3. **Clear file name**
   English filename that explains what the file is.

Example:

```text
GS_Logo_Primary_Gold_Print.ai
GS_Logo_Primary_Gold_Print_Preview.png
```

## 5. Digital vs Print

### Digital Assets

Used for:

* Website.
* Social media.
* Mockups.
* Reels.
* Customer previews.

Digital assets may include soft pink details, shadows, and subtle visual effects.

---

### Print Assets

Used for:

* Outer gift bag.
* Inner box.
* Sticker.
* Gift card.
* Tissue paper.
* Packaging production.

Print assets must follow:

```text
Single-color muted / satin gold print
```

Print assets should not include:

* Gradients.
* Shadows.
* Multiple colors.
* Low-resolution images.
* Very thin lines.
* Crowded details.

---

## 6. Customer-Facing Image Rules

Customer-facing images must not show:

* Price tags.
* Discount stickers.
* Retail source names.
* Ross or outlet labels.
* MSRP.
* Dollar signs.
* Purchase information.
* Messy backgrounds.
* Personal information.

Customer-facing images should be:

* Clean.
* Premium.
* Soft.
* Clear.
* Suitable for website, commerce platform, or social media.

---

## 7. File Naming Rules

Use English filenames only.

Good examples:

```text
GS_LittleStories_BabySet_Pink_6M_ClientDisplay_v1.png
GS_Box_Inner_40x30x10_Mockup_v1.png
GS_Logo_Primary_Digital_v1.svg
```

Avoid:

```text
final final logo.ai
image123.png
new design copy.pdf
gift story 2 copie.pdf
```

## 8. Asset Status

Use these labels when reviewing assets:

```text
Approved
Print Ready
Digital Only
Preview Only
Needs Review
Needs Export
Archived
```

No asset should be printed unless it is marked as:

```text
Print Ready
```

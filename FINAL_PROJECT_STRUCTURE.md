# Gift Story — Final Project Structure

## 1. Purpose

This file is the final checklist for the clean Gift Story project structure.

Use this file before uploading the project to GitHub or sharing it with a designer, developer, or operator.

The goal is to make sure all important files are organized and nothing is scattered.

## 5. docs File Roles

### 00_MASTER_SOURCE.md

The top-level approved source of truth for Gift Story.

### 01_FOUNDATION_DECISIONS.md

Core decisions about brand, website, commerce, pricing, packaging, WhatsApp, and launch direction.

### 02_BRAND_POSITIONING.md

Brand positioning, promise, tone, personality, and differentiation.

### 03_CATEGORIES_AND_DROPS.md

Gift category system and Limited Drops structure.

### 04_PACKAGING_SYSTEM.md

Packaging components, dimensions, print rules, protection, and quality checks.

### 05_PRICING_MODEL.md

Cost-based pricing model, 50% profit-above-cost rule, channel fees, and pricing tiers.

### 06_COMMERCE_CHANNELS.md

Role of the official website, Salla/Zid, delivery apps, email, forms, and WhatsApp.

### 07_WEBSITE_STRUCTURE.md

Official website pages, sections, CTA, and website strategy.

### 08_ASSETS_INDEX.md

Visual asset organization, naming, digital/print rules, and preview requirements.

### 09_OPERATIONS_AND_POLICIES.md

Order operations, support, delivery, policies, returns, packaging protection, and logistics.

### 10_LAUNCH_PLAN.md

Phased launch plan, priorities, checklists, and soft launch rules.

Gift Story logo files.

Should include:

```text

```

### castle/

Castle illustration files for story identity and packaging.

### pattern/

Stars, moon, dots, and tissue/background pattern files.

### sticker/

Round sticker files for tissue closure.

### cards/

Gift card design files and previews.

### packaging/

Box, outer gift bag, packaging mockups, protection references.

Approved sizes:

```text

Outer gift bag: 45 × 33 × 15 cm
```

### product-photos/

Recommended structure:

```text

      edited/
      client-display/
```

### mockups/

Visual mockups for website, packaging, and presentations.

### social/

Instagram, TikTok, story templates, reels covers, and launch visuals.

### video/

Raw and edited videos.

### references/

Inspiration, old references, supplier images, screenshots, or non-approved visual references.

## 21. archive Folder Rules

The archive folder is for old, duplicate, unclear, or superseded files.

Archive files are not active source files.

Use archive only for:

* Historical reference.
* Recovering old ideas.
* Comparing previous versions.
* Checking original concepts.

Do not use archive files directly for website, packaging, pricing, or customer-facing content unless reviewed and re-approved.

## 24. What Should Not Be in the Root

Do not leave these in the root folder:

```text
random PDFs
old screenshots
raw product photos
duplicate logo files
old HTML files
unapproved assets
supplier images
temporary exports
final final copy files
```

Move them to:

```text
archive/
```

or the correct active folder if approved.

---

## 25. File Naming Rules

Use English filenames.

Good:

```text
GS_Logo_Primary_Gold_Print.ai
GS_Box_Inner_40x30x10_Mockup.png
GS_LittleStories_BabySet_Pink_6M_ClientDisplay_v1.png
```

Avoid:

```text
final final logo.ai
copy new design.pdf
gift story 2 copie.pdf
IMG_1234.jpg
new file latest.png
```

---

## 26. Before GitHub Upload

Before first upload:

1. Remove duplicate files.
2. Move old files to archive.
3. Make sure all markdown files are saved.
4. Make sure Excel file is inside inventory.
5. Make sure no private customer data is included.
6. Make sure raw product images are not used as customer-facing images.
7. Make sure official files are easy to understand.
8. Make sure folder names are lowercase where possible.
9. Make sure the project can be opened easily in VS Code.
10. Make sure README.md explains the project clearly.

---

## 27. Final Project Rule

The project folder should feel like an operating system for Gift Story.

Anyone opening it should understand:

* What Gift Story is.
* What is currently active.
* What is coming later.
* How pricing works.
* How packaging works.
* How website content is structured.
* How commerce should be handled.
* How special requests are reviewed.
* Where assets belong.
* What is active and what is archive.

If a file does not have a clear role, move it out of the active source structure.

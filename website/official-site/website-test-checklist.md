# Gift Story — Website Test Checklist

## 1. Purpose

This checklist is used to review the Gift Story official website before uploading it to GitHub and publishing it on Cloudflare Pages.

The goal is to make sure the website is:

* Clear.
* Premium.
* Arabic-first.
* Mobile-friendly.
* Aligned with Gift Story.
* Free from old pricing or WhatsApp-first flow.
* Ready to connect to the official commerce platform.

---

## 2. File to Test

Main file:

```text
index.html
```

Open it locally in the browser before uploading.

---

## 3. First Visual Check

Open the website and check the first impression.

The website should feel:

```text
Soft
Premium
Warm
Clean
Elegant
Spacious
Gift-focused
```

The website should not feel:

```text
Crowded
Cheap
Discount-focused
Like a random store
Like a basic product catalog
Like a WhatsApp-only order page
```

---

## 4. Brand Check

Confirm the following appears correctly:

```text
Gift Story
قفت ستوري
WHERE EVERY GIFT TELLS A STORY
```

Brand direction should be clear:

```text
A premium gifting experience
Limited Drops
Gift stories
Curated packaging
Little Stories as first active category
```

---

## 5. Language and Direction Check

Confirm:

```text
Arabic is the primary language
Page direction is RTL
Arabic text is readable
English is used only as support
Buttons are clear
No awkward translation appears
```

---

## 6. Visual Identity Check

Approved visual direction:

```text
Off-white
Creamy beige
Nude
Muted / satin gold
Soft warm dark text
```

Check that the site does not use:

```text
Dark navy as main identity
Bright colors
Harsh black sections
Neon colors
Strong discount colors
Crowded graphics
Cartoon style
```

---

## 7. Header Check

Header should include clear navigation.

Check:

```text
Logo or brand name appears clearly
Navigation links work
Main CTA appears
Header is not crowded
Header works on mobile
```

Main CTA should be:

```text
تسوّقي القصة الحالية
```

Do not use WhatsApp as the main header CTA.

---

## 8. Hero Section Check

Hero should immediately explain Gift Story.

Confirm hero includes:

```text
Gift Story
Slogan
Premium gifting idea
Clear main CTA
Special request secondary CTA
Soft visual direction
```

Hero should not include:

```text
Fixed product price
Shopping cart
Raw product inventory
WhatsApp-first ordering
Heavy sales language
```

---

## 9. Main CTA Check

Main CTA:

```text
تسوّقي القصة الحالية
```

Expected destination:

```text
https://shop.gift-story.com
```

or the approved Salla / Zid shop link once ready.

If the shop is not ready yet, the link may remain temporary, but it must be updated before public launch.

---

## 10. Little Stories Section Check

Confirm the first active category is clear:

```text
هدايا الصغار
Little Stories — Baby & Kids Gifts
```

The section should explain:

```text
Soft baby and kids gifts
Curated gift stories
Ready-to-gift experience
Limited availability
```

Do not show other categories as active products.

---

## 11. Pricing Check

The official website should not show detailed active product prices by default.

Confirm there is no old fixed price such as:

```text
299 ريال
```

unless intentionally shown in a specific approved context.

The website should direct customers to the official shop for current prices and availability.

---

## 12. Cart / Checkout Check

The official website should not include:

```text
Cart
Add to cart
Checkout form
Payment form
Inventory system
Product quantity selector
```

These belong to Salla, Zid, or the approved commerce platform.

---

## 13. WhatsApp Check

WhatsApp should not be the main order flow.

Official WhatsApp number if shown:

```text
+966576362810
```

It may appear only as secondary support.

Do not show the logistics number publicly:

```text
0576239046
```

This number is internal only.

---

## 14. Packaging Section Check

Confirm packaging is explained as part of the value.

Must include:

```text
Outer gift bag
Inner gift box
Tissue paper
Sticker
Gift card
Protective packaging when needed
```

Approved sizes:

```text
Inner box: 40 × 30 × 10 cm
Outer gift bag: 45 × 33 × 15 cm
```

Protective packaging clarification:

```text
Protective nylon or shipping carton is logistics protection only, not part of the premium unboxing identity.
```

---

## 15. Print / Color Note Check

The website should include this note or equivalent:

```text
قد تختلف بعض التفاصيل اللونية بين العروض الرقمية والتغليف المطبوع، حيث تعتمد Gift Story في التغليف الفعلي على طباعة فاخرة بلون ذهبي موحد حسب مواصفات الإنتاج.
```

This note protects against confusion between digital mockups and actual packaging.

---

## 16. Previous Stories Check

Previous Stories should feel like inspiration, not active inventory.

Confirm wording communicates:

```text
قصص سابقة من Gift Story تُعرض كإلهام فقط وقد لا تتكرر بنفس التفاصيل.
```

Do not make previous stories purchasable unless they are actually available in the commerce platform.

---

## 17. Coming Soon Check

Coming Soon categories may include:

```text
قصص العناية
قصص التخرج
قصص أعياد الميلاد
قصص من السفر
```

Check that they are not shown as active products.

Do not add prices or order buttons for Coming Soon categories.

---

## 18. Special Request Check

Special Request section should explain:

```text
Custom requests are reviewed
Submission is not automatic approval
Execution depends on availability, time, and feasibility
Price is confirmed after review
```

Important line:

```text
إرسال الطلب لا يعني تأكيده مباشرة. تتم مراجعة الطلب أولًا حسب التوفر وإمكانية التنفيذ.
```

CTA should lead to the form or special request section.

---

## 19. Policies Check

Homepage should include a clear policy summary.

Must include:

```text
لا يمكن الاسترجاع أو الاستبدال بعد خروج الطلب من مقر التجهيز، إلا في حال وجود خطأ واضح من Gift Story أو حسب ما تقتضيه الأنظمة.
```

Also confirm these ideas appear:

```text
Special requests require review
Some content details may vary based on availability
Protective packaging may be used for delivery or shipping
```

---

## 20. Contact Check

Contact section should include professional paths.

Allowed:

```text
Shop link
Special request link
Email placeholder
Instagram placeholder
Secondary WhatsApp support
```

Do not make logistics contact public.

---

## 21. Image Check

If images are included, confirm they do not show:

```text
Price tags
Store labels
Discount stickers
Ross
MSRP
Dollar signs
Supplier screenshots
Messy backgrounds
Personal information
```

Images should look:

```text
Clean
Soft
Premium
Gift-ready
Aligned with Gift Story colors
```

---

## 22. Mobile Check

Test the website on mobile width.

Confirm:

```text
Header is readable
Buttons are easy to tap
Sections stack properly
Text is not too small
Images do not overflow
No horizontal scrolling
CTA is visible
Page feels spacious
```

Mobile is more important than desktop for launch.

---

## 23. Desktop Check

Test desktop view.

Confirm:

```text
Layout is balanced
Sections are not too wide
Text blocks are comfortable
Images have enough space
CTA buttons are visible
Footer is clean
```

---

## 24. Link Check

Click every link.

Check:

```text
Header navigation links work
Shop CTA works
Special request CTA works
Policy link works
Contact link works
Footer links work
No empty button exists
No broken anchor link exists
```

Temporary links should be clearly marked before launch and replaced when ready.

---

## 25. SEO Check

Confirm basic metadata exists:

```text
Page title
Meta description
Arabic language attribute
Open Graph title
Open Graph description
Open Graph image placeholder or approved image
```

Suggested title:

```text
Gift Story | هدايا فاخرة تحكي قصة
```

Suggested meta description:

```text
Gift Story براند سعودي للهدايا المنسقة بعناية، يبدأ بهدايا الصغار وتجربة تغليف فاخرة حيث كل هدية تحكي قصة.
```

---

## 26. Performance Check

Before publishing, confirm:

```text
Page loads fast
Images are not extremely large
No unnecessary scripts
No heavy animation
CSS is simple
No broken assets
```

---

## 27. Content Cleanup Check

Before upload, remove:

```text
Placeholder text
Draft comments visible to customers
Old Baby Story Box-only copy
Old 299 fixed price copy
Old WhatsApp-first order language
Old cart language
Random English text
Repeated sections
Internal notes
```

---

## 28. Final Pre-Upload Checklist

The website is ready to upload when:

```text
Brand name is correct
Slogan is correct
Arabic RTL works
No fixed wrong price appears
No cart appears
No WhatsApp-first ordering appears
Shop CTA is clear
Special request CTA is clear
Packaging sizes are correct
Policy summary is visible
Logistics number is not public
Images are clean
Mobile view is good
All links work
Footer is complete
```

---

## 29. Final Decision

The website can be uploaded when it clearly does this:

```text
Build trust, explain Gift Story, show the current story direction, explain packaging value, and guide customers to shop or request a special story.
```

If the site tries to sell like a random store, revise it before launch.
